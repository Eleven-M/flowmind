# FlowMind 1.4.5

FlowMind 1.4.5 narrows the product around one reliable path: route a request to a skill, execute it through a real adapter or MCP-compatible provider, capture feedback, and reuse that feedback on the next similar run.

## Highlights

- Fixed routing order so explicit `--skill` execution wins before generic learning detection
- Injected active FlowMind runtime and current skill context into `learning-feedback`
- Repaired `resource-bind list` so it reads providers through a stable registry API
- Upgraded `log-audit` from a planning stub to a real adapter execution path when a provider is configured
- Improved TUI focus handling so sidebar/chat input and status display stay aligned

## Verification

- Added regression tests for routing order, learning bindings, registry listing, and executable skill behavior
- Re-ran core tests successfully before release: `9/9` passing
- Published npm package: `flowmind@1.4.5`

## Try It

```bash
npm install -g flowmind
flowmind skills --json
flowmind process --skill log-audit "查询 traceId abc123 的日志" --json
flowmind "下次日志结果用表格格式"
flowmind process --skill log-audit "查询 traceId def456 的日志" --json
```
